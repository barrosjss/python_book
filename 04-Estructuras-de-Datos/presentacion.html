01
Listas [ ]
La estructura de datos más usada en Python

📋 Lista list

Colección ordenada y mutable de elementos. Puede contener cualquier tipo de dato, incluso mezclas. Es la estructura más flexible de Python.

# Crear listas
numeros   = [1, 2, 3, 4, 5]
nombres   = ["Ana", "Luis", "María"]
mixta     = [1, "dos", 3.0, True, None]
vacia     = []

# Acceder por índice
print(nombres[0])    # "Ana"
print(nombres[-1])   # "María" — último

# Slicing
print(numeros[1:4])  # [2, 3, 4]
print(numeros[::-1]) # [5, 4, 3, 2, 1] invertida

# Modificar elemento
nombres[1] = "Carlos"  # ✓ las listas son mutables

Operaciones básicas:

# Longitud
print(len(numeros))       # 5

# Verificar existencia
print(3 in numeros)       # True
print(99 not in numeros)  # True

# Concatenar listas
a = [1, 2]
b = [3, 4]
print(a + b)              # [1, 2, 3, 4]

# Repetir
print([0] * 5)            # [0, 0, 0, 0, 0]

# Funciones útiles
print(sum(numeros))        # 15
print(max(numeros))        # 5
print(min(numeros))        # 1

02
Tuplas ( )
Como una lista, pero que no se puede modificar

🔒 Tupla tuple

Colección ordenada e inmutable. Una vez creada, no puedes cambiar sus elementos. Perfecta para datos que no deben modificarse.

# Crear tuplas
punto     = (10, 20)
colores   = ("rojo", "verde", "azul")
mixta     = (1, "dos", True)
un_elem   = (42,)   # coma obligatoria para 1 elemento

# Acceder — igual que lista
print(punto[0])     # 10
print(colores[-1])  # "azul"

# Desempaquetar
x, y = punto
print(x, y)         # 10 20

# Intentar modificar → Error
# punto[0] = 99     # TypeError!

Características

  • Ordenada — tiene índices
  • Inmutable — no cambia
  • Permite duplicados
  • Más rápida que list

Solo 2 métodos

  • .count(x) — cuántas veces aparece
  • .index(x) — posición de x

¿Cuándo usar tuple en vez de list?

  • Coordenadas, colores RGB: (255, 128, 0)
  • Datos de configuración que no cambian
  • Claves de diccionario (los dict no aceptan list como clave)
  • Cuando quieres proteger datos de modificaciones accidentales
list → tuple: tuple([1,2,3])
tuple → list: list((1,2,3))
03
Diccionarios { }
Pares clave-valor — la base de JSON y los objetos

🗝️ Diccionario dict

Colección de pares clave: valor. Accedes a los valores por clave, no por índice. Mutable desde Python 3.7 mantiene el orden de inserción.

# Crear diccionario
usuario = {
    "nombre": "Carlos",
    "edad":   30,
    "activo": True
}

# Acceder por clave
print(usuario["nombre"])    # "Carlos"
print(usuario.get("edad"))  # 30 — forma segura

# Agregar / modificar
usuario["email"] = "ana@example.com"
usuario["edad"]  = 31

# Eliminar
del usuario["activo"]
# Recorrer claves y valores
for clave in usuario.keys():
    print(clave)

for valor in usuario.values():
    print(valor)

for clave, valor in usuario.items():
    print(f"{clave}: {valor}")
[ ] vs .get()
usuario["telefono"] → KeyError si no existe
usuario.get("telefono") → None sin error
usuario.get("telefono", "N/A") → valor por defecto

04
Conjuntos { , }
Sin orden, sin duplicados — ideal para operaciones de conjuntos

🔵 Set (Conjunto) set

  • Sin duplicados — valores únicos automáticamente
  • Sin orden — no se accede por índice
  • Mutable — puedes agregar o quitar elementos
  • Operaciones matemáticas de conjuntos incluidas
# Crear set
frutas  = {"manzana", "pera", "uva"}
numeros = {1, 2, 2, 3, 3}  # → {1, 2, 3}

# Set vacío — IMPORTANTE: no usar {}
vacio = set()              # {} crea un dict vacío!

# Agregar / quitar
frutas.add("mango")
frutas.discard("pera")    # no lanza error si no existe

# Truco: eliminar duplicados de lista
lista = [1, 2, 2, 3, 3, 3]
unicos = list(set(lista))   # [1, 2, 3]

Operaciones matemáticas de conjuntos:

a = {1, 2, 3}
b = {3, 4, 5}

print(a | b)    # Unión:          {1,2,3,4,5}
print(a & b)    # Intersección:   {3}
print(a - b)    # Diferencia:     {1, 2}
print(a ^ b)    # Dif. simétrica: {1,2,4,5}
MétodoDescripción
.add(x)Agrega un elemento
.remove(x)Elimina; error si no existe
.discard(x)Elimina; sin error si no existe
.issubset(b)¿a es subconjunto de b?
.isdisjoint(b)¿No tienen elementos en común?

¿Cuál usar y cuándo?

Usa list cuando…

  • Necesitas orden y modificar
  • Permites duplicados
  • Recorres con for
  • Es tu opción por defecto

Usa tuple cuando…

  • Datos no deben cambiar
  • Coordenadas, configs
  • Clave de diccionario
  • Necesitas rendimiento

Usa dict cuando…

  • Datos tienen nombre (clave)
  • Trabajas con JSON/APIs
  • Describes un "objeto"
  • Búsqueda rápida por nombre

Usa set cuando…

  • Necesitas valores únicos
  • Eliminar duplicados
  • Unión / intersección
  • El orden no importa
# Ejemplo práctico — perfil de usuario
usuario = {                                # dict — tiene propiedades con nombre
    "nombre":     "Ana García",
    "hobbies":    ["leer", "correr", "cocinar"],   # list — orden importa, puede cambiar
    "coordenadas":(4.7110, -74.0721),              # tuple — no cambia
    "permisos":   {"leer", "escribir"}               # set — únicos, sin orden
}
