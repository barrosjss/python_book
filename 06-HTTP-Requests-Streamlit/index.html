Sección 1

HTTP vs HTTPS

Cuando abres una página web, tu compu le pide datos a otro servidor. Para que se entiendan, usan un protocolo común.

🔓 HTTP

  • HyperText Transfer Protocol
  • Datos viajan en texto plano
  • En WiFi pública: cualquiera puede interceptar y leer tu petición con herramientas como Wireshark
  • Puerto por defecto: 80

🔒 HTTPS

  • HTTP + SSL/TLS (encriptado)
  • Negocia una clave secreta entre cliente y servidor → todo cifrado
  • El 🔒 del navegador = certificado SSL que verifica identidad del servidor
  • Puerto por defecto: 443
💡
Para el programador

Python con requests maneja HTTPS automáticamente — verifica el certificado y cifra todo. Tú solo escribes la URL con https://. Si alguna vez necesitas deshabilitar la verificación (solo en pruebas): requests.get(url, verify=False).

Sección 1

Ciclo Request / Response

Todo se reduce a un vaivén de dos mensajes

1

📤 Request — Tu compu → Servidor

Contiene: método (GET/POST/…), URL (qué recurso pides), headers (metadatos como Content-Type, Authorization) y body (solo en POST/PUT — los datos que envías).

2

📥 Response — Servidor → Tu compu

Contiene: código de estado (200, 404, 500…), headers de respuesta (tipo de contenido, cookies) y body (los datos JSON que pediste).

🖥️  →→→ request →→→  🌐
Tu compu                            Servidor
🖥️  ←←← response ←←←  🌐

Sección 2

Anatomía de una URL

Una URL es la dirección exacta de un recurso en internet. Cada parte tiene su función.

https://rickandmortyapi.com:443/api/character/1?name=rick&status=alive
Esquema (protocolo) Dominio (servidor) Puerto Path (ruta del recurso) Query parameters (filtros)
ParteEjemploPara qué sirve
Esquemahttps://Protocolo de comunicación
Dominiorickandmortyapi.comDirección del servidor
Path/api/character/1Recurso específico que pides
Query params?name=rick&status=aliveFiltros adicionales (siempre empieza con ?)
Sección 2

Query Parameters — Los filtros

Van después del ? en la URL. Separa múltiples params con &

/api/character?name=rick

→ Busca personajes con nombre "rick"

/api/character?status=alive&species=human

→ Personajes vivos Y humanos (dos filtros con &)

/api/character?page=2

→ Segunda página de resultados

🎯
Regla

El path dice QUÉ recurso quieres. Los query params dicen CÓMO lo quieres filtrado o paginado.

One does not simply

usar GET para crear datos en el servidor

Para eso existe POST 👇 — cada acción tiene su método

Sección 5 — Librería requests

requests — GET básico

$ pip install requests
Python
import requests

# 1. Hacer la petición GET
r = requests.get("https://rickandmortyapi.com/api/character/1")

# 2. Ver el código de estado
print(r.status_code)   # 200  ← ¡todo bien!

# 3. Obtener los datos como diccionario Python
personaje = r.json()

print(personaje["name"])    # "Rick Sanchez"
print(personaje["status"])  # "Alive"
print(personaje["species"]) # "Human"
print(personaje["image"])   # URL de la imagen
🎯
Qué tiene el objeto respuesta

r.status_code · r.json() · r.text · r.headers · r.url · r.elapsed

🔄
¿Por qué .json() funciona?

El servidor manda texto (un string de JSON). r.json() llama internamente a json.loads(r.text) — lo parsea y devuelve un dict Python. Si el servidor respondió HTML en vez de JSON (ej: error 500 de Django), .json() lanza excepción. Ver r.text primero para diagnosticar.

Sección 5 — Librería requests

requests — GET con filtros

Python
import requests

# En vez de escribir la URL a mano con ?name=rick&status=alive
# pasamos los params como diccionario → requests arma la URL
params = {
    "name": "rick",
    "status": "alive"
}

r = requests.get(
    "https://rickandmortyapi.com/api/character",
    params=params
)
# URL real que se construye: /api/character?name=rick&status=alive

data = r.json()

print(f"Total encontrados: {data['info']['count']}")
print(f"Páginas: {data['info']['pages']}")

personajes = data["results"]   # lista de personajes
for p in personajes[:3]:
    print(p["name"], "—", p["status"])
Sección 5 — Librería requests

requests — POST y manejo de errores

POST — Enviar datos

Python
import requests

nuevo = {
    "title": "Mi post",
    "body": "Aprendiendo APIs",
    "userId": 1
}

r = requests.post(
    "https://jsonplaceholder.typicode.com/posts",
    json=nuevo   # ← json= convierte dict a JSON Y agrega
                 #   Content-Type: application/json al header
)

print(r.status_code)  # 201 Created (no 200: se creó algo nuevo)
print(r.json())       # servidor responde con el objeto + su ID
⚠️
json= vs data=

data=dict envía como formulario HTML (nombre=Rick&edad=70). json=dict envía como JSON ({"nombre":"Rick"}). Las APIs REST esperan JSON — siempre usa json=.

Manejo de errores

Python
import requests

r = requests.get(
    "https://rickandmortyapi.com/api/character/9999"
)

# Opción 1: verificar manual
if r.status_code == 200:
    datos = r.json()
elif r.status_code == 404:
    print("Personaje no existe")

# Opción 2: automático (recomendado)
try:
    r.raise_for_status()  # lanza excepción si 4xx o 5xx
    datos = r.json()
except requests.exceptions.HTTPError as e:
    print(f"Error: {e}")
💡
¿Por qué raise_for_status?

requests no falla automáticamente en errores 4xx/5xx — solo falla si no puede conectarse. raise_for_status() convierte cualquier respuesta de error en una excepción Python para que tu código no procese silenciosamente una respuesta fallida.

Sección 6

Streamlit + requests = Poder real

app.py
import streamlit as st
import requests

st.title("🛸 Rick and Morty — Vista previa")

nombre = st.text_input("Nombre del personaje", "Rick")

if st.button("Buscar"):
    with st.spinner("Consultando la API del Rickverse..."):
        r = requests.get(
            "https://rickandmortyapi.com/api/character",
            params={"name": nombre}
        )

    if r.status_code == 200:
        data = r.json()
        personajes = data["results"]
        st.success(f"¡Encontré {data['info']['count']} personaje(s)!")

        cols = st.columns(3)
        for i, p in enumerate(personajes[:3]):
            with cols[i]:
                st.image(p["image"])
                st.write(f"**{p['name']}**")
                st.write(p["status"])

    elif r.status_code == 404:
        st.error("¡Ese personaje no existe en ninguna dimensión! 🌀")

Paso 1 — Estructura base y búsqueda

Paso 1 / 3
rick_morty_app.py
import streamlit as st
import requests

st.set_page_config(
    page_title="Rick & Morty Explorer",
    page_icon="🛸",
    layout="wide"
)

st.title("🛸 Rick and Morty Explorer")
st.caption("Powered by rickandmortyapi.com")

# --- BÚSQUEDA POR NOMBRE ---
st.header("🔍 Buscar Personaje")

nombre = st.text_input(
    "Nombre del personaje",
    placeholder="Ej: Rick, Morty, Beth, Jerry..."
)

if st.button("Buscar", type="primary"):
    with st.spinner("Viajando por las dimensiones..."):
        r = requests.get(
            "https://rickandmortyapi.com/api/character",
            params={"name": nombre}
        )

    if r.status_code == 200:
        data = r.json()
        st.success(f"¡Wubba Lubba Dub Dub! {data['info']['count']} personaje(s) encontrado(s)")
    elif r.status_code == 404:
        st.error("¡Ese personaje no existe en ninguna dimensión! 🌀")

Paso 3 — Perfil completo por ID

Paso 3 / 3

rick_morty_app.py — agregar al final
st.divider()
st.header("🆔 Perfil Completo de Personaje")

char_id = st.number_input("ID del personaje (1 - 826)", min_value=1, max_value=826, value=1)

if st.button("Ver perfil completo"):
    r = requests.get(f"https://rickandmortyapi.com/api/character/{char_id}")

    if r.status_code == 200:
        p = r.json()
        col1, col2 = st.columns([1, 2])
        with col1:
            st.image(p["image"], use_container_width=True)
        with col2:
            st.title(p["name"])
            estado = "🟢 Vivo" if p["status"] == "Alive" else \
                     "🔴 Muerto" if p["status"] == "Dead" else "❓ Desconocido"
            st.metric("Estado", estado)
            st.write(f"**Especie:** {p['species']}")
            st.write(f"**Género:** {p['gender']}")
            st.write(f"**Origen:** {p['origin']['name']}")
            st.write(f"**Ubicación actual:** {p['location']['name']}")
            st.write(f"**Episodios:** {len(p['episode'])}")

        # Mostrar la URL que construimos (¡educativo!)
        with st.expander("🔗 Ver URL de la petición"):
            st.code(f"GET https://rickandmortyapi.com/api/character/{char_id}")
    else:
        st.error(f"Personaje {char_id} no encontrado")
